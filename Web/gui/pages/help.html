How do I add a job to the cluster?
  1. Select the Jobs menu in the left panel.
  2. Click the Add button in the top header.
  3. Fill out the Add Job form:
    • GitHub URL: web address of the job's source code.
    • Local File: local address of the job's source code.
    • Name: user specified name for the job.
    • Priority: the rank of the job on the cluster.
      • Lower numbers denote higher priority for faster job completion.
      • Higher numbers denote lower priority for slower job completion.
    • Tasks per Bundle: tasks per network message of the job.
      • Increase Tasks per Bundle for more speed and less updates.
      • Decrease Tasks per Bundle for less speed and more updates.
  4. Click the ADD button to add the job to the Jobs table.
  • The added job will immediately begin running on the cluster.
How do I remove a job from the cluster?
  1. Select the Jobs menu in the left panel.
  2. Check the job(s) to be removed.
  3. Click the Remove button in the top header.
  • The checked job(s) are now removed from the Jobs table.
What does the status of a job mean?
  • Verify: the server is verifying the job's tasks and source code.
  • Build: the server is sending the job's tasks to the clients to be built.
  • Idle: the job is at rest and is not progressing towards completion.
  • Active: the job is running and progressing towards completion.
  • Paused: the job is paused and not progressing towards completion.
  • Done: the job has finished running and is now completed.
How do I add a unit to the cluster?
  1. Select the Units menu in the left panel.
  2. Click the Add button in the top header.
  3. Fill out the Add Unit form:
    • Name: user specified name for the unit.
    • IP Address: network address of the connected unit.
    • Username: user account on the connected unit.
    • Password: user password of account on the connected unit.
    • Cores Limit: sets limit on the number of active cpu cores on the unit.
      • Increase Cores Limit for more resources and more load on unit.
      • Decrease Cores Limit for less resources and less load on unit.
  4. Click the ADD button to add unit to Units table.
  • The added unit will immediately increase cluster resources and run jobs.
How do I remove a unit from the cluster?
  1. Select the Units menu in the left panel.
  2. Check the unit(s) to be removed.
  3. Click the Remove button in the top header.
  • The checked unit(s) are now removed from the Units table.
What does the status of a unit mean?
  • Online: the unit is connected and being used on the cluster.
  • Offline: the unit is not connected and not being used on the cluster.
How do I start, pause, or stop a job?
  1. Select the Jobs menu in the left panel.
  2. View the Controls column and the row of the desired job:
    • Start: will run or resume the job from an idle state when clicked.
    • Pause: will suspend the job from a running state when clicked.
    • Stop: will return the job to the initial state when clicked.
      • CAUTION! Results of the job will be cleared when stopped.
How do I start, pause, or stop all jobs?
  • View left side of interface above the Download button.
How do I get the results of a job?
  1. Select the Jobs menu from the left panel.
  2. View the Results column and the row of the desired job.
  3. Click the Ready button.
  • The job's results will be download to user's local computer.
